# UI + Testing: Tailwind + shadcn/ui + Vitest + Playwright

UI primitives: Tailwind CSS (utility-first, tokens Atelier Zero mapeados em `tailwind.config`) + shadcn/ui (componentes copiáveis, não lib black-box — Button, Card, Toast ficam no repo, versionados, customizáveis). shadcn encaixa porque a anatomia visual do wireframe é específica e tokens-driven — não dá pra usar Material UI ou Chakra sem override massivo. Testing: Vitest pra service-level (state machine da sala, validação Zod, handlers WebSocket) + Playwright pra E2E multi-client (dois contexts de browser validam reveal em tempo real). Alternativas: CSS Modules (mais código), MUI (muito opinionated), Jest (mais lento), Cypress (multi-client mais verboso). Trade-off: shadcn/ui exige "copiar código" no repo em vez de `npm install` — versionamento manual, mas ganho em controle. Playwright exige browser binaries instaladas (Chromium/Firefox/WebKit) — setup inicial de ~200MB.
