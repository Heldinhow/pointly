# Interações: Arremessos Sincronizados e Cooldown

Para adicionar micro-interações lúdicas na mesa (arremessar tomates, café, papel) sem gerar caos e mantendo consistência visual absoluta para todos os participantes. Decidimos centralizar a lógica no servidor: os Arremessos possuem um Cooldown de 5 segundos controlado client-side (e validado server-side), e o Desfecho (acerto, desvio ou rebote) é sorteado pelo backend e transmitido via WebSocket para garantir que todos vejam a mesma animação e reações correspondentes simultaneamente. Alternativas: processar desfechos no client de quem atira (geraria inconsistência para os outros) ou permitir disparo livre (poluiria a interface e aumentaria o tráfego do servidor desnecessariamente). Trade-off: pequena latência de rede no sorteio do desfecho do arremesso, mas que é absorvida pela própria física em arco da animação do Projétil.
